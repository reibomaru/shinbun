# 仕様書: AI / Web開発 Daily 情報アグリゲータ

---

## 1. 概要

### 1.1 プロダクト名（仮）
AI Tech Daily Aggregator

### 1.2 目的
- 生成AIおよびWeb開発領域の最新アップデートを取り逃さない
- 日々登場する有用ライブラリ・ツールの情報を体系的に蓄積する
- 日経新聞の一面のように、Dailyで全体像を俯瞰できる体験を提供する

### 1.3 利用形態
**個人利用専用**。マルチユーザー・外部サービス提供は対象外。

### 1.4 成功指標（KPI）
- 毎日のダイジェストで「重要なアップデートを見逃さない」状態を実現
- 1日10分以内で主要トピックを把握できる
- 主要ツールのChangelog取り逃しゼロ
- LLM処理コストが月額予算内に収まる（目標: $30以下/月）

---

## 2. スコープ

### 2.1 対象領域
- 生成AI（LLM、エージェント、ツール、推論最適化など）
- Web開発（フロントエンド / バックエンド / DevTools）
- グローバル情報（英語・日本語中心）

### 2.2 対象情報源

#### Phase 1（MVP）
| 情報源 | ポーリング間隔 | 優先度 |
|---|---|---|
| GitHub Releases（監視対象リポジトリ） | 1時間 | 必須 |
| RSS（Zenn / Qiita / はてな / 企業ブログ） | 1時間 | 必須 |
| 主要ツールの公式Changelog | 6時間 | 必須 |
| Hacker News（Top / Best） | 30分 | 必須 |

#### Phase 2
| 情報源 | ポーリング間隔 | 優先度 |
|---|---|---|
| YouTubeチャンネル（開発系） | 6時間 | 推奨 |
| Reddit（r/MachineLearning, r/webdev） | 1時間 | 推奨 |
| arXiv（cs.AI, cs.CL） | 12時間 | 推奨 |

#### Phase 3
- X（API制約のため後回し）
- Notion連携

### 2.3 多言語対応方針
- 英語記事 → 日本語で要約を生成
- 日本語記事 → 日本語のまま要約
- ダッシュボード表示言語: 日本語
- 原文リンクは常に保持

## 3. ユーザージャーニー

### 3.1 典型的な1日の流れ

```
朝 8:00 ─────────────────────────────────────────────────────────
  Slack に Daily Digest が届く
  スマホでざっと確認（5分）
    ├─ Top 5記事をスキャン
    ├─ 気になる記事に 🔖 リアクション（後で読む）
    └─ 明らかに不要な通知に 👎（フィードバック）

日中 ────────────────────────────────────────────────────────────
  必要に応じて Slack 通知を確認
    ├─ 🚨 緊急アラート（セキュリティ / Breaking Change）
    │    → 即時通知のため業務中でも見落とさない
    └─ 特になければスルーでOK

夜・すき間時間 ──────────────────────────────────────────────────
  Webダッシュボードを開く（5〜10分）
    ├─ 未読フィルタで当日未確認の記事を確認
    ├─ カテゴリ別セクションで興味分野を深読み
    ├─ 気になる記事を保存 + タグ付け
    └─ Changelog列で追っているツールの更新を確認

週次 ────────────────────────────────────────────────────────────
  保存一覧を見返す（5分）
    └─ タグで整理・不要なものを削除
```

### 3.2 情報との接点マップ

| 状況 | 接点 | 期待される体験 |
|---|---|---|
| 朝の通勤・起床後 | Slack Daily Digest | 流し読みで今日の重要情報を把握 |
| 業務中の緊急確認 | Slack 緊急アラート | 見落とせないものだけ割り込んでくる |
| まとまった時間がある時 | Webダッシュボード | 一面を眺めて気になるものを深読み |
| 情報をストックしたい時 | 保存機能 | タグ付けして後から参照できる |
| 特定ツールの更新を追う時 | Changelog列 / ウォッチリスト | 見逃さず追跡できる |

### 4.5 設定（YAML管理）

設定画面は提供しない。ウォッチリスト・情報源・配信設定はすべてリポジトリ内の YAML ファイルを直接編集して管理する。

```
config/
├── sources.yaml    # 情報源リスト
├── watchlist.yaml  # ウォッチリスト
└── settings.yaml   # 配信設定
```

## 5. 機能要件

### 5.1 情報収集

| 要件 | 内容 |
|---|---|
| 自動取得 | 設定された情報源から定期的に新着を取得する |
| フィルタリング | AI/Web開発に関連しない記事は除外する |
| 重複排除 | 複数ソースで同一記事が取得された場合は1件にまとめる |
| 要約生成 | 取得した記事の日本語要約を自動生成する |
| 重要度判定 | ソースの信頼性・鮮度・反応数をもとにスコアを算出する |
| 緊急度判定 | セキュリティアドバイザリ・Breaking Changeを自動検出する |

### 5.2 Slack通知

| 要件 | 内容 |
|---|---|
| Daily Digest | 毎朝設定時刻（デフォルトJST 8:00）にTop N件を配信する |
| カテゴリハイライト | GenAI / Frontend / Backend / Tools の件数をサマリーで通知する |
| 緊急アラート | 緊急度が高い記事はDigestを待たず即時通知する |
| フィードバック | 各記事への👍👎🔖リアクションを記録する |
| ダッシュボードリンク | ダッシュボードへの導線をメッセージに含める |

### 5.3 Webダッシュボード

| 要件 | 内容 |
|---|---|
| アクセス制限 | パスワード認証により本人のみアクセスできる |
| 一面表示 | Top Stories / カテゴリ別 / Releases の3ペインレイアウト |
| 未読管理 | クリック済み記事をグレーアウトし「未読のみ表示」フィルタを提供する |
| 詳細表示 | 要約・Key Points・Why it matters・関連記事を表示する |
| フィードバック | 各カードにフィードバックボタンを表示し記録する |

### 5.4 保存機能

| 要件 | 内容 |
|---|---|
| 保存 | 記事をワンクリックで保存できる（Slackリアクションからも可） |
| 一覧表示 | 保存一覧を日付順・カテゴリ別に表示する |
| タグ付け | 自由入力タグを付与できる。LLMラベルから候補を提示する |
| エクスポート | Markdown形式でエクスポートできる（Phase 2） |

### 5.5 ウォッチリスト

| 要件 | 内容 |
|---|---|
| 登録対象 | エンティティ（ライブラリ・企業）またはキーワードを `watchlist.yaml` に記述する |
| スコアブースト | ウォッチリスト対象の記事は重要度スコアを優先的に引き上げる |
| 緊急通知 | 対象ごとに緊急通知の ON/OFF を YAML で設定する |
| 優先表示 | Daily Digest とダッシュボードで TOP 表示される |

### 5.6 Changelog監視

| 要件 | 内容 |
|---|---|
| 初期設定 | Claude Code を必須として `sources.yaml` に含める |
| 追加方法 | `sources.yaml` を直接編集して追加する |
| 専用表示 | ダッシュボードの右カラムに Changelog 専用列を設ける |

## 6. システム設計概略

### 6.1 コンポーネント構成

```
┌──────────────────────────────────────────────────────────────┐
│  外部情報源                                                   │
│  （GitHub / RSS / Hacker News / YouTube / arXiv 等）          │
└─────────────────────┬────────────────────────────────────────┘
                      │ 定期取得（ポーリング）
                      ▼
┌──────────────────────────────────────────────────────────────┐
│  収集・処理エンジン                                           │
│                                                              │
│  ① 取得  →  ② フィルタ  →  ③ AI分類・要約  →  ④ スコアリング│
│                                                              │
│  ・重複記事を除外                                            │
│  ・無関係な記事を除外                                        │
│  ・日本語で要約を生成                                        │
│  ・重要度スコアを算出                                        │
│  ・緊急記事を即時検出                                        │
└────────────────┬──────────────────────────────┬─────────────┘
                 │ 保存                          │ 緊急時即時送信
                 ▼                              ▼
┌───────────────────────────┐      ┌────────────────────────────┐
│  データベース             │      │  Slack 緊急アラート         │
│  （記事・要約・スコア）    │      │  （セキュリティ / Breaking） │
└───────────────┬───────────┘      └────────────────────────────┘
                │
       ┌────────┴─────────┐
       │ 毎朝              │ 常時
       ▼                  ▼
┌──────────────┐  ┌────────────────────────────────────────────┐
│ Slack        │  │  Webダッシュボード                         │
│ Daily Digest │  │  一面 / 詳細 / 保存 / 設定                 │
└──────────────┘  └────────────────────────────────────────────┘
```

### 6.2 処理の流れ（概略）

1. **取得**: 定期的に各情報源から新着記事を取得する
2. **フィルタ**: 重複・無関係な記事をルールベースで除外する
3. **AI分類**: 関連性・緊急度・カテゴリをAIで判定する
4. **要約生成**: 重要度が高い記事の日本語要約をAIで生成する
5. **スコアリング**: ソース信頼度・鮮度・反応数・AIの品質評価から重要度スコアを算出する
6. **保存**: 処理済み記事をDBに蓄積する
7. **配信**: 毎朝まとめてSlackへ送信。緊急記事は即時送信する

## 7. MVP範囲（フェーズ定義）

### Phase 1a: Slack-first MVP
最小限で「Slackに届く情報が役に立つか」を検証する。Webダッシュボードは後回し。

- [ ] RSS + GitHub Releases の自動取得
- [ ] Hacker News の自動取得
- [ ] AI による関連記事フィルタ・要約・スコアリング
- [ ] Daily Slack通知（Top N + カテゴリ別）
- [ ] 緊急アラート（セキュリティ / Breaking Change）
- [ ] Claude Code Changelog監視
- [ ] Slackリアクション（👍👎🔖）の記録
- [ ] LLMコスト追跡とアラート

### Phase 1b: Webダッシュボード
- [ ] 一面レイアウト（Top / カテゴリ別 / Changelog列）
- [ ] 記事詳細表示
- [ ] 既読管理
- [ ] 保存機能（一覧 + タグ付け）
- [ ] パスワード保護

### Phase 2: 情報拡充・検索
- [ ] YouTube / Reddit / arXiv 追加
- [ ] 意味検索（ベクトル検索）

### Phase 3: 連携・自動化
- [ ] X連携
